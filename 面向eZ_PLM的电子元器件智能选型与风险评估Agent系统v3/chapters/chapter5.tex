\section{实验设计}
实验验证围绕需求解析正确性、器件推荐合理性和风险识别准确性展开，使用 28 条测试用例在 LLM 增强模式下运行完整 Pipeline。具体设计如下：
\begin{itemize}
    \item \textbf{单元级解析测试：}针对四级容错解析链设计 15 种典型句式变体，验证 mA 单位转换、否定句式处理、温度多格式匹配及电压方向修正等关键路径。

    \item \textbf{端到端选型评测：}运行完整 Pipeline，以 JSON 记录解析结果、推荐数量、各维度评分、风险等级及时延，并与预设期望逐字段比对。

    \item \textbf{eZ-PLM API 集成测试：}在配置有效密钥的环境下验证 HMAC-SHA256 签名鉴权、多前缀分组查询和 MPN 电压推断等关键链路。

    \item \textbf{测试用例构成：}28 条用例中，Buck 14 条（5\,V--48\,V 输入，0.5\,A--6\,A 输出，覆盖工业/车规/国产偏好），Boost 4 条（3.3\,V$\rightarrow$5\,V、5\,V$\rightarrow$12\,V 等），LDO 10 条（1.2\,V--5\,V 输出，0.1\,A--2\,A，含车规与国产需求）。

    \item \textbf{运行模式：}通过 DeepSeek-V3 完成需求解析和 LLM 评分，结合 eZ-PLM API 实时检索与参考设计拉取，验证真实推理链路下的端到端性能。
\end{itemize}

\section{eZ-PLM 实时 API 集成测试}

以"12\,V 转 5\,V，3\,A，工业级"为输入执行端到端查询，系统遍历 9 个制造商前缀、发起 9 次 API 请求后返回 31 条 TI 在产器件（TPS54020RUWR、TPS54220DDAR、LM2596S 系列等），封装覆盖 WSON、HTSSOP、D/N 等多种形态。五维过滤后全部有效器件输入电压满足 4.5$\sim$36\,V、输出电流 $\ge$ 3\,A、温度覆盖 $-40\sim85\;$°C，验证了 HMAC-SHA256 签名鉴权、多前缀分组查询和 MPN 电压推断链路的端到端正确性。

\section{局限性分析}

当前评测仍存在三方面局限：第一，LLM 增强评分依赖 eZ-PLM 参考设计数据，受平台覆盖范围限制，部分器件无法获取关联参考设计而回退至纯规则模式，导致 LLM 软评分维度未能覆盖全部候选器件；第二，三类工程文档目前通过人工定性检查验证，尚未建立规范覆盖率、公式准确率等自动化评测指标；第三，API 实时测试中部分器件的库存字段为空值，系统将其供应链分设为缺省中性值 50，该处理方式无法充分反映真实供货状态。
