% 中文摘要
% ============================================================
电子元器件选型是硬件产品研发中劳动密集度高、风险集中度高的关键环节。传统人工流程要求工程师在众多制造商提供的海量型号中逐一核查多维参数，因而通常耗时较长，且难以实现供应链风险的系统化识别与标准化工程文档的快速生成。本文面向硬禾科技 eZ-PLM 电子元器件管理平台，设计并实现了一套基于大语言模型（Large Language Model, LLM）与工程规则引擎协同推理的智能选型与风险评估智能体系统。该系统采用“自然语言需求解析—实时 API 器件检索—混合智能评分—证据链构建—标准化报告生成”的五阶段流水线式处理架构，并集成基于 LangChain 框架的 ReAct 推理行动交互模式，以支持多轮对话与动态决策。

系统的主要技术贡献体现在五个方面：第一，针对 eZ-PLM 平台支持型号前缀检索的特点，提出制造商型号段分组查询策略，使单次查询能够召回 200 条以上在产物料；第二，设计由 LLM 语义解析、规则覆盖、电压比较兜底和温度正则匹配构成的四级容错需求解析链，解决参数单位误识别、否定句式漏判、LLM 拓扑幻觉等六类工程问题；第三，构建以 eZ-PLM 参考设计数据为上下文约束的 LLM 混合评分机制，实现参数匹配、供应链稳定性、应用适配度与设计成熟度四个维度的加权评分，并在缺少参考设计数据时自动降级为基于规则的双维评分；第四，建立规则引擎事实验证与 LLM 文本生成相分离的双层风险评估架构，覆盖 13 类确定性风险检测，以保障风险报告的严谨性和数据可追溯性；第五，定义 PartIR、ScoreBreakdown、TopologyIR、RiskIR 等标准化中间表示，生成物料清单（Bill of Materials, BOM）、风险评估报告和电路拓扑分析报告三类工程文档。

实验结果表明，系统在 28 条测试用例上实现了 92.9\% 的通过率，核心参数字段解析准确率达 100\%；结合检索增强生成（Retrieval-Augmented Generation, RAG）工程知识库与 ReAct 智能体交互模式，系统的知识覆盖深度与交互灵活性得到了进一步提升。
