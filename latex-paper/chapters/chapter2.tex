\section{总体架构设计}
\label{sec:architectrue}
系统采用“服务层—流水线推理层—工具层—数据层”的四层架构，将自然语言需求输入到三类工程文档输出的全过程组织为可自动执行的处理链路。各层之间职责边界清晰，数据流转方向限定为自上而下的逐层加工过程，从而提升系统的模块化程度、可维护性和可扩展性。

服务层基于 FastAPI 框架构建 API 服务，对外提供以下五个核心端点：\texttt{GET /health} 用于系统健康检查；\texttt{POST /analyze} 用于接收自然语言需求并返回完整的 \texttt{SelectionReport} JSON；\texttt{POST /replacement} 用于接收原始器件型号并返回 \texttt{ReplacementReport} JSON；\texttt{POST /agent/chat} 用于支持携带 \texttt{session\_id} 的多轮对话式选型交互；\texttt{GET /agent/sessions} 用于查看当前活跃的 Agent 会话列表。该层同时负责请求路由、参数校验、UTF-8 中文编码兼容和异常捕获，保障前端与推理层之间通信的稳定性。

流水线推理层实现了以下五阶段线性处理流程，将各阶段输出作为下一阶段输入，从而完成从用户意图到工程文档的端到端自动化转换。
\begin{itemize}
    \item 在需求解析阶段，自然语言文本经四级容错解析链转换为结构化约束对象。该过程首先由 LLM 完成语义映射，再由规则引擎通过关键词覆盖、电压方向校验和温度正则匹配逐级修正潜在错误;
    \item 在器件检索阶段，系统以结构化约束为输入，优先通过 HMAC-SHA256 签名认证调用 eZ-PLM 实时 API 执行多前缀分组查询，并使用五维过滤函数筛选满足约束的候选器件；当 API 不可用或返回空结果时，系统自动回退到离线器件数据库;
    \item 在混合评分阶段，系统对候选器件执行双模自适应评分：当检测到当前器件存在 eZ-PLM 关联参考设计时，启用四维增强评分模式；否则自动降级为纯规则双维评分模式。两种模式共享参数评分和供应链评分两个基础维度，仅在总分合并权重上有所差异，从而保证评分结果的可比性和可降级性;
    \item 在证据构建阶段，系统为每个候选器件生成 \texttt{EvidenceIR} 证据链条目，标注数据来源、计算置信度，并自动标记需要人工复核的异常项，例如库存为零、生命周期状态为停产或数据手册缺失等;
    \item 在报告生成阶段，评分结果与证据链被整合为 \texttt{SelectionReport}，并进一步生成 BOM 选型清单、供应链风险评估报告和电路拓扑分析报告三类工程文档，同时输出结构化 \texttt{TopologyIR} JSON 文件，以支持与 eZ-PLM 平台的数据互通。
\end{itemize}

工具层封装了六类可被上层调用的核心能力模块：eZ-PLM API 客户端、离线器件数据管理器、混合评分引擎、证据构建器、报告生成器和替代器件查找器。其中，eZ-PLM API 客户端负责 HMAC-SHA256 签名生成、请求发送和响应解析；离线器件数据管理器负责 JSON 格式器件库的加载与过滤；混合评分引擎集成规则评分与 LLM 软评分的双模切换逻辑；证据构建器负责八类字段级证据的自动生成；报告生成器分别面向 BOM、风险评估和拓扑分析三类文档；替代器件查找器则基于 \texttt{replacement\_for} 字段和同类拓扑交叉匹配，实现国产替代和功能等效替代搜索。上述模块均可独立测试和替换，有助于降低系统耦合度。

数据层汇聚了三类异构数据源。第一类为 eZ-PLM API，包括物料查询接口和参考设计查询接口。第二类为本地离线器件数据库，以 JSON 格式存储 1237 条器件记录，覆盖 Buck、Boost 和 LDO 三类拓扑，并包含国产与进口制造商、工业级与车规级等多维标签；当 API 不可用时，该数据库作为兜底数据源保障系统的独立运行能力。第三类为基于 ChromaDB 向量存储的 RAG 工程知识库，已收录 29 条工程知识条目，覆盖 Buck/Boost/LDO 设计公式、热管理、车规认证、PCB Layout 规范、供应链评估框架、EMI/EMC 设计、可靠性降额准则和 USB-C PD 协议等 11 个专业类别。知识条目通过 \texttt{sentence-transformers/all-MiniLM-L6-v2} 嵌入模型转换为 384 维向量，并在每次 Pipeline 分析过程中自动检索，检索结果随后被注入三类输出报告。

\section{评分模型设计}
\subsection{双模自适应评分框架}

为兼顾评分结果的稳定性与智能化程度，系统设计了双模自适应评分框架。评分模式的选择取决于两个条件：一是 LLM 服务是否可用，二是候选器件是否具备 eZ-PLM 参考设计数据。仅当两项条件同时满足时，系统启用 LLM 增强模式；否则自动回退至纯规则模式。该机制既能够利用参考设计知识提升评分质量，又保证了在外部模型或数据不可用时系统仍具备稳定运行能力。

\subsection{纯规则模式}

当 LLM 不可用或缺少参考设计数据时，系统采用基于参数匹配和供应链稳定性的双维评分模型：

\begin{equation}
S_{\text{total}} = 0.80 \times S_{\text{param}} + 0.20 \times S_{\text{supply}}
\label{eq:rule}
\end{equation}

上述权重的设定依据了电子元器件选型领域的多属性决策实践。在元器件选型与管理的经典方法论中，电气参数的匹配程度始终被视作决定器件可用性的首要约束——若输入电压范围、输出电流能力或工作温度等级等硬性指标不满足设计要求，则该器件在工程上不可用，因此参数匹配维度的权重被赋予了主导地位（0.80）\cite{pecht2008parts}。供应链稳定性的权重（0.20）则参照了供应链风险评估领域中对库存水平与生命周期状态两大类指标的综合赋权研究\cite{chen2021supplyrisk,goel2021supply}，在参数匹配主导的前提下为供应风险的量化留出了合理的表达空间。

参数匹配分 $S_{\text{param}}$ 采用动态约束归一化方法计算：

\begin{equation}
S_{\text{param}} = \frac{\displaystyle\sum_{i=1}^{N} s_i}{N} \times 100
\label{eq:param}
\end{equation}

式中，$N$ 表示实际存在的参数约束项数量，$s_i \in [0,1]$ 表示对应约束项的归一化得分。当用户未提供明确参数约束时，系统将参数匹配分设为中性值 50，以避免对候选器件产生系统性惩罚。

\subsection{LLM 增强模式}

当 LLM 服务可用且候选器件具备参考设计数据时，评分模型由双维规则评分扩展为四维混合评分：

\begin{equation}
S_{\text{total}} = 0.40 S_{\text{param}} + 0.10 S_{\text{supply}} + 0.25 S_{\text{app}} + 0.25 S_{\text{design}}
\label{eq:hybrid}
\end{equation}

其中，$S_{\text{param}}$ 和 $S_{\text{supply}}$ 分别表示参数匹配分和供应链稳定性分，$S_{\text{app}}$ 和 $S_{\text{design}}$ 分别表示应用场景适配度和设计成熟度。与纯规则模式相比，增强模式将参数匹配权重由 0.80 调整为 0.40，将供应链稳定性权重由 0.20 调整为 0.10，并将释放出的 0.50 权重分配给两个基于参考设计数据的 LLM 软评分维度。该设计遵循“硬约束主导、软知识增强”的分层评估思路 \cite{wang2022selection}：在参考设计数据可用时，引入知识增强维度以提升应用适配性判断能力；在参考设计数据缺失或 LLM 输出异常时，自动回退至纯规则模式，以保证评分稳定性。

为降低 LLM 输出不确定性对评分结果的影响，系统将 $S_{\text{app}}$ 和 $S_{\text{design}}$ 的取值范围限定为 $[0,100]$，并对模型返回结果进行格式校验与数值边界检查。若返回结果格式不完整、字段缺失或分值越界，系统将触发回退机制，改用纯规则评分结果。

\subsection{去重、排序与推荐分级}
eZ-PLM API 返回同一型号族的查询结果时，可能包含多个仅在封装后缀、质量等级或卷带标识上存在差异的变体。以 TI 的 LM2596 系列为例，查询结果可能同时包含 LM2596S-5.0/NOPB、LM2596SX-5.0/NOPB 和 LM2596T-5.0/LF03 等记录。这些器件在核心电气特性上高度一致，主要差异体现在封装形式或包装方式。若直接全部纳入推荐列表，容易造成信息冗余，并降低不同型号之间的有效区分度。

为减少同型号族变体对排序结果的干扰，系统通过 \texttt{\_extract\_core\_mpn()} 函数提取候选器件的核心 MPN（Manufacturer Part Number，即制造商型号）。该过程会去除 “/NOPB”“/LF03” 等后缀、“-TR”“-T\&R” 等卷带标识，以及末尾的非功能性封装变体字符，从而获得代表器件电气型号的核心标识。随后，系统以核心 MPN 为键对候选器件进行分组去重，每组仅保留综合得分最高的变体，并将被合并变体的完整型号信息记录至 \texttt{reasons} 字段，以保留结果可追溯性。

去重完成后，系统按照综合得分降序排序，并采用 Top-N 分级策略生成推荐结果。综合得分不低于 75 分且排名前五的器件被标注为“推荐”（recommended）；排名第六至第十五且综合得分不低于 50 分的器件被标注为“备选”（backup）；其余器件则标注为“不推荐”（not\_recommended）。最终输出结果最多保留 15 条记录，以兼顾推荐结果的信息完整性和前端展示的可读性。

（四）\texttt{RiskIR} 描述选型方案的整体风险及其结构化风险条目。整体风险等级划分为低（low）、中（medium）和高（high）三级，用于表征方案的供应链与工程风险水平。每条风险条目（\texttt{RiskItem}）均由规则引擎生成，包含风险类型、严重程度、风险描述、相关器件型号、缓解建议及证据引用字段（\texttt{evidence\_refs}）。该结构遵循失效模式与影响分析（Failure Mode and Effects Analysis, FMEA）的“失效模式—失效影响—失效原因—现行控制措施—建议措施”记录范式，使风险信息能够追溯至其触发条件和处置路径。\texttt{evidence\_refs} 字段为后续引入风险优先级数（Risk Priority Number, RPN）提供支持。RPN 由严重度（Severity）、发生频率（Occurrence）和可检测度（Detection）三项指标乘积计算得到，可在现有定性风险分级基础上进一步实现定量评估。在当前实现中，规则引擎已覆盖 13 类确定性风险场景，包括无候选器件、库存不足、停产器件、供应商过度集中以及关键字段缺失等。所有风险检测结果均以 \texttt{RiskItem} 列表形式统一输出，LLM 仅负责补充工程背景说明，不参与风险判定或新增风险条目，从而保证风险结论建立在可验证的事实基础之上。


（五）\texttt{SelectionReport} 与 \texttt{ReplacementReport} 分别作为流水线选型分析和替代器件搜索的顶层输出结构。\texttt{SelectionReport} 封装了原始用户输入、解析后的结构约束、候选器件完整列表、推荐器件子集、风险汇总、证据链全集和摘要文本；\texttt{ReplacementReport} 则封装了原始型号信息、替代候选列表、兼容等级评估结论和结构化的对比摘要，为国产替代和功能等效替代两类场景提供了统一的数据返回格式。
