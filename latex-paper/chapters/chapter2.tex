\section{总体架构设计}

系统采用“展示层—服务层—推理层—工具层—数据层”五层架构：

\textbf{展示层}：基于Streamlit构建Web界面，提供自然语言输入、选型结果展示和Agent推理步骤可视化。

\textbf{服务层}：基于FastAPI构建RESTful API，暴露三个核心端点：\texttt{POST /analyze}（Pipeline模式，返回SelectionReport JSON）、\texttt{POST /agent/chat}（Agent模式，支持多轮对话）、\texttt{POST /replacement}（替代器件查找）。

\textbf{推理层}：提供双模推理引擎——Pipeline模式为五阶段线性流水线（需求解析→器件检索→多维评分→证据构建→报告生成），适用于批量处理；Agent模式基于LangChain 1.3 \texttt{create\_agent} API，配置search\_components、query\_design\_knowledge、find\_alternative\_parts、generate\_full\_report四个工具，通过“思考→行动→观察”循环自主编排调用顺序。

\textbf{工具层}：封装系统的四个关键能力——元器件搜索（eZ-PLM API HMAC签名+mock兜底）、工程知识检索（ChromaDB语义检索）、替代器件查找（replacement\_for字段交叉匹配）、完整报告生成（BOM+风险评估+拓扑分析+TopologyIR JSON）。

\textbf{数据层}：eZ-PLM API（HMAC-SHA256签名认证，物料+参考设计查询）、ChromaDB向量库（12条工程知识，all-MiniLM-L6-v2嵌入）、Mock数据（209条本地器件，国产/进口、车规/非车规、Buck/Boost/LDO全覆盖）。

\section{评分模型设计}

系统采用多维度加权评分，支持双模式自适应切换。

\subsection{纯规则模式}

当LLM不可用时回退至纯规则模式：
\begin{equation}
S_{\text{total}} = 0.80 \times S_{\text{param}} + 0.20 \times S_{\text{supply}}
\end{equation}
其中$S_{\text{param}}$基于参数检查数动态归一化：$S_{\text{param}} = \frac{\sum s_i}{N} \times 100$（$N$为实际检查项数，$s_i \in [0,1]$为单项得分）。$S_{\text{supply}}$基于库存水平和生命周期状态分段计算。

\subsection{LLM增强模式}

当LLM可用且有参考设计数据时，启用四维加权：
\begin{equation}
S_{\text{total}} = 0.40 S_{\text{param}} + 0.10 S_{\text{supply}} + 0.25 S_{\text{app}} + 0.25 S_{\text{design}}
\end{equation}
新增$S_{\text{app}}$（应用场景适配度）和$S_{\text{design}}$（设计成熟度）由LLM结合eZ-PLM参考设计库数据综合评估。

\subsection{去重与推荐分级}

针对API返回的多封装变体（如LM2596S-5.0/NOPB与LM2596SX-5.0/NOPB），通过核心型号提取函数进行去重合并。推荐分级采用Top-N策略：Top 5为推荐（$\geq$75分），6-15为备选（50-74分），其余不推荐。

\section{IR数据模型设计}

为满足eZ-PLM平台的数据互通要求，定义了三种标准化中间表示：

\textbf{PartIR}（23字段）：覆盖制造商信息、电气参数、封装、认证等级、库存价格、生命周期状态等完整器件描述。针对固定输出电压器件设计了MPN推断函数，避免API“最大输出电压”范围值被误用。

\textbf{RiskIR}：包含整体风险等级和风险条目列表。每条RiskItem记录风险类型（供应链/工程技术）、严重程度、描述和缓解建议。系统内置13条检测规则，覆盖无候选器件、库存水平、生命周期、供应商集中度、数据完整性、地缘政治合规、车规认证等维度。扩展FMEA RPN量化分析（严重度$\times$发生概率$\times$检测难度，1-5分制）。

\textbf{TopologyIR}（4子模型）：TopoNode（拓扑功能节点，8-10个，含邻接关系）、ExternalComponent（外围元件，7-8个，含推荐值和选型备注）、ThermalEstimate（热估算：效率/功耗/结温/散热判断）、顶层TopologyIR整合上述子模型+Mermaid框图+Layout规则+设计引用。通过\texttt{\_detect\_converter\_type()}自动识别集成式Converter与外置MOSFET Controller，生成差异化节点拓扑。
