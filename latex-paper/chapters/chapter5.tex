\section{实验设计}
系统的实验验证围绕需求解析正确性、器件推荐合理性及风险识别准确性展开。实验使用 28 条覆盖多类别、多拓扑、多约束组合的测试用例，并在 LLM 增强模式下运行完整 Pipeline 流程。具体设计如下：
\begin{itemize}
    \item \textbf{单元级解析测试：}针对四级容错需求解析链设计 15 种典型句式变体，重点验证 mA 单位转换、否定句式处理、温度范围多格式匹配及电压方向修正等关键解析路径的正确性。

    \item \textbf{端到端选型评测：}通过自动化评测框架运行完整 Pipeline 流程，并以 JSON 格式记录解析结果、推荐器件数量、各维度评分、风险等级及运行时延。随后，将系统输出与预设期望结果进行逐字段比对，以检验解析、检索、评分、证据构建与报告生成五个阶段的准确性与一致性。

    \item \textbf{eZ-PLM API 集成测试：}在配置有效 eZ-PLM API 密钥的环境下执行实时查询，验证 HMAC-SHA256 签名鉴权、多前缀分组查询、API 物料字段解析及 MPN 固定电压推断等关键链路的正确性。

    \item \textbf{测试用例构成：}28 条测试用例包括 14 条 DC-DC Buck 降压转换器用例、4 条 DC-DC Boost 升压转换器用例和 10 条 LDO 低压差线性稳压器用例。其中，Buck 用例覆盖 5\,V--48\,V 输入电压、0.5\,A--6\,A 输出电流，以及工业级、车规级和国产替代偏好；Boost 用例覆盖 3.3\,V$\rightarrow$5\,V、5\,V$\rightarrow$12\,V 等常见升压场景；LDO 用例覆盖 1.2\,V--5\,V 输出电压、0.1\,A--2\,A 输出电流，并包含车规级与国产替代需求。

    \item \textbf{运行模式：}系统通过 DeepSeek-V3 完成自然语言需求解析，通过 eZ-PLM API 实时检索在产物料并获取参考设计案例，再利用 LLM 对候选器件的应用场景适配度及设计成熟度进行评分，从而验证 Pipeline 在真实推理链路下的端到端性能。
\end{itemize}

\section{风险检测与证据链分析}

风险等级分布方面，28 条用例中 7 条被评定为 LOW 风险，17 条被评定为 MEDIUM 风险，4 条被评定为 HIGH 风险，整体呈现出以中低风险为主体、高风险场景相对集中的分布特征。LOW 风险用例通常具备候选器件数量充足、库存水平较健康且生命周期状态为在产等特征；MEDIUM 风险用例多由推荐数量偏少、供应商集中度较高或部分数据字段缺失等因素触发；HIGH 风险则主要集中在约束组合较为严格的场景，例如同时要求车规认证、国产品牌及特定电压电流组合时，满足全部条件的候选器件数量有限，规则引擎会据此输出较高等级的风险提示。

\section{关键工程缺陷发现与修复}

系统在评测迭代过程中，识别并修复了多类关键工程缺陷，以提升选型准确性和可靠性。具体包括：
\begin{enumerate}
    \item \textbf{固定输出电压器件误推荐}：如 \texttt{LM2596S-12}（输出 12\,V）在 5\,V 需求中被错误包含。根因是 API 返回的“最大电压范围 40\,V”被误认为可调器件。修复措施是在过滤阶段引入 MPN 电压推断，并结合 $\pm 5\%$ 容差匹配执行精确筛选。
    \item \textbf{LLM 对电压方向误判}：典型例子为“5\,V 转 3.3\,V”被错误识别为 LDO。通过添加电压比较方向的兜底规则（三级校正），确保正确识别为 Buck 降压器件。
    \item \textbf{mA 单位量级误差}：部分输入如“输出 500\,mA”被误识别为 500\,A。修复方案为引入 mA 单位前置正则表达式，将数值正确转换为安培，消除 1000 倍量级错误。
    \item \textbf{否定句式解析失败}：例如“非车规”未被正确映射为工业级。通过增加上下文否定词检测，实现对否定语义的准确解析。
    \item \textbf{LLM 返回等级字段未归一化}：如“车规级”“AEC-Q100”等表述未统一为标准 \texttt{automotive} 值。修复方案是增加等级归一化映射表，实现统一标准化处理。
    \item \textbf{口语化降压表述未触发类别识别}：诸如“降到 5\,V”的自然语言未被识别为 DC-DC 触发词。通过扩充关键词覆盖列表，实现对口语化表达的正确类别识别。
\end{enumerate}

上述缺陷修复后，所有回归测试均通过验证，并在随后的迭代中未出现新的回归问题，确保了系统选型结果的稳定性与可靠性。

\section{eZ-PLM 实时 API 集成测试}

在配置有效 API 密钥的环境下，以"12\,V 转 5\,V，3\,A，工业级"为输入，系统执行了端到端查询。在系统共遍历了 9 个制造商型号前缀（TPS54、TPS62、LM2596 等），发起 9 次 API 请求后，成功返回 31 条来自德州仪器（TI）在产器件——型号涵盖 TPS54020RUWR、TPS54220DDAR、LM2596S 系列等，封装覆盖了 WSON、HTSSOP 和 D/N 等多种形态。五维过滤后全部有效器件的输入电压范围均满足 4.5$\sim$36\,V 的要求，输出电流均不低于 3\,A，温度范围均覆盖 $-40\sim85\;$°C，验证了 HMAC-SHA256 签名鉴权、多前缀分组查询和 MPN 电压推断链路的端到端正确性。

\section{局限性分析}

当前评测仍存在若干局限。首先，在评分维度方面，LLM 增强评分依赖 eZ-PLM 平台中的参考设计案例数据，但受平台数据覆盖范围限制，部分器件无法获取关联参考设计。对于此类器件，系统会自动回退至纯规则双维评分，导致应用适配度与设计成熟度两个 LLM 软评分维度未能覆盖全部候选器件。

其次，在报告质量评估方面，三类工程文档目前主要通过人工定性检查进行验证，尚未建立自动化评测指标，例如规范覆盖率、公式计算准确率和风险描述一致性等。因此，报告生成质量仍需要在后续工作中通过更系统的量化指标进行评估。

最后，在供应链数据方面，API 实时测试中部分器件的库存字段为空值，即 API 未返回 \texttt{stock} 数据。系统当前将此类器件的供应链分设为缺省中性值 50，但该处理方式无法充分反映真实库存水平和供货状态，可能影响供应链稳定性评分的精确性。
