\section{测试方案设计}

系统测试按三个层次递进展开：

\begin{enumerate}[label=(\arabic*)]
  \item \textbf{单元测试}：验证需求解析器对15种典型中英文句式的解析准确性；
  \item \textbf{端到端评测}：10个DC-DC Buck选型用例，覆盖常规/车规/宽压/高功率/国产替代场景，验证Pipeline全链路正确性；
  \item \textbf{Agent功能测试}：验证单轮选型和多轮追问场景的工具调用效率与输出质量。
\end{enumerate}

\subsection{关键Bug修复验证}

评测过程中发现并修复了多个影响选型准确性的关键Bug：

\textbf{Bug 1——固定电压器件误推荐}：修复前，dc\_dc\_005（需求Vout=3.3V）的TOP1为LM2596S-12/NOPB（固定12V输出），完全偏离需求。修复后正确推荐TPS1100D（可调输出），Vout匹配度从0\%提升至100\%。

\textbf{Bug 2——非车规误判}：修复前，“非车规”被解析为grade=automotive，导致dc\_dc\_003所有候选被车规认证条件过滤。修复后正确解析为grade=industrial，dc\_dc\_003恢复为1推荐。

\textbf{Bug 3——温度解析失败}：修复前，“-40到85°C”仅匹配“85°C”（因-40后无°C后缀），导致Tmin=85, Tmax=None。修复后正确解析为Tmin=-40, Tmax=85。

\section{Agent多轮对话测试}

以“12V转5V 3A 车规级降压方案”为初始需求，测试Agent在单轮和多轮场景下的表现：

\textbf{单轮测试}：
\begin{itemize}
  \item 工具调用：2次（search\_components → query\_design\_knowledge）
  \item 总耗时：约8秒（含DeepSeek API调用和RAG检索）
  \item 输出质量：返回包含LM2596S-5.0推荐方案的结构化对比表，引用RAG知识库中的Buck选型要点验证输入耐压余量
\end{itemize}

\textbf{多轮追问测试}（“第一个推荐方案有国产替代吗？对比一下”）：
\begin{itemize}
  \item 工具调用：4次（search\_components → query\_design\_knowledge → find\_alternative\_parts → search\_components）
  \item Agent正确记住了第一轮推荐的LM2596S-5.0/NOPB型号
  \item 返回了进口vs国产的对比分析表，包含输出电流余量、温度范围、车规认证等维度的差异说明
\end{itemize}

\textbf{已知限制}：当Mock数据库中没有国产替代型号时，Agent偶尔出现幻觉编造型号（如XL2596S）。后续将通过validate-before-reply中间件缓解此问题。
