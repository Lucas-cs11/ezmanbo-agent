\section{实验设计}
系统的实验验证围绕需求解析正确性、器件推荐合理性及风险识别准确性展开。实验使用 28 条覆盖多类别、多拓扑、多约束组合的测试用例，并在 LLM 增强模式下运行完整 Pipeline 流程。具体设计如下：
\begin{itemize}
    \item \textbf{单元级解析测试：}针对四级容错需求解析链设计 15 种典型句式变体，重点验证 mA 单位转换、否定句式处理、温度范围多格式匹配及电压方向修正等关键解析路径的正确性。

    \item \textbf{端到端选型评测：}通过自动化评测框架运行完整 Pipeline 流程，并以 JSON 格式记录解析结果、推荐器件数量、各维度评分、风险等级及运行时延。随后，将系统输出与预设期望结果进行逐字段比对，以检验解析、检索、评分、证据构建与报告生成五个阶段的准确性与一致性。

    \item \textbf{eZ-PLM API 集成测试：}在配置有效 eZ-PLM API 密钥的环境下执行实时查询，验证 HMAC-SHA256 签名鉴权、多前缀分组查询、API 物料字段解析及 MPN 固定电压推断等关键链路的正确性。

    \item \textbf{测试用例构成：}28 条测试用例包括 14 条 DC-DC Buck 降压转换器用例、4 条 DC-DC Boost 升压转换器用例和 10 条 LDO 低压差线性稳压器用例。其中，Buck 用例覆盖 5\,V--48\,V 输入电压、0.5\,A--6\,A 输出电流，以及工业级、车规级和国产替代偏好；Boost 用例覆盖 3.3\,V$\rightarrow$5\,V、5\,V$\rightarrow$12\,V 等常见升压场景；LDO 用例覆盖 1.2\,V--5\,V 输出电压、0.1\,A--2\,A 输出电流，并包含车规级与国产替代需求。

    \item \textbf{运行模式：}系统通过 DeepSeek-V3 完成自然语言需求解析，通过 eZ-PLM API 实时检索在产物料并获取参考设计案例，再利用 LLM 对候选器件的应用场景适配度及设计成熟度进行评分，从而验证 Pipeline 在真实推理链路下的端到端性能。
\end{itemize}

\section{风险检测与证据链分析}

风险等级分布方面，28 条用例中 7 条被评定为 LOW 风险，17 条为 MEDIUM 风险，4 条为 HIGH 风险，呈现出以中低风险为主体的合理分布格局。LOW 风险用例的典型特征是候选器件数量充足（5 条以上推荐）、库存水平健康且生命周期状态均为在产；MEDIUM 风险用例通常因单一供应商集中度偏高或推荐数量略少而被标注为中等风险；HIGH 风险集中于少数约束组合较为苛刻的场景——例如同时要求车规认证、国产品牌和特定电压电流组合时，满足全部约束的候选器件数量有限，风险引擎据此正确地输出了"推荐器件不足"的高等级风险。这种分布特征表明，规则引擎能够根据候选器件的覆盖程度和供应链状态如实区分不同场景的风险水平，而非一律给出虚假的低风险评估。表~\ref{tab:risk_dist}和图~\ref{fig:risk_dist}分别给出了风险等级分布的数值和可视化展示。

在 LLM 增强模式下，28 条用例的端到端延迟中位数为 4.55 秒，均值为 4.89 秒。其中延迟最大的用例（dc\_dc\_001，约 16.5 秒）执行了完整的 eZ-PLM API 多前缀查询链路——包含 9 次前缀查询和多次参考设计接口调用——其延迟主要来自网络 I/O 而非本地计算。其余用例的延迟集中在 2$\sim$8 秒区间，主要由 LLM 推理调用（需求解析和候选器件评分）和 eZ-PLM API 器件检索两部分构成。考虑到每次分析涉及 LLM 语义解析、多前缀 API 检索、LLM 四维评分、证据链构建和三份报告生成等多个环节，平均 5 秒以内的端到端延迟相较于传统人工选型流程（通常耗时数小时）已实现了数量级的效率提升，完全能够满足工程师的实时交互需求。图~\ref{fig:latency}展示了各用例的延迟分布。

\section{关键工程缺陷发现与修复}

在评测迭代过程中，系统性地发现并修复了多类影响准确性的工程缺陷。一是固定电压器件误推荐——LM2596S-12（输出 12\,V）出现在 5\,V 需求的结果中，根因在于 API 属性返回"最大电压范围 40\,V"被误认为可调器件，修复方案是在过滤阶段调用 MPN 电压推断并执行 $\pm5\%$ 容差匹配；二是"5\,V 转 3.3\,V"被 LLM 误判为 LDO，通过添加电压比较方向兜底规则（第三级）加以修正；三是 mA 单位量级误差，添加 mA 前置正则后消除了 1000 倍的量级错误；四是否定句式解析失败，增加上下文否定词检测后正确地把"非车规"映射为工业级；五是 LLM 返回的等级字段未归一化——"车规级""AEC-Q100"等表述未被映射到标准的 \texttt{automotive} 值，通过增加等级归一化映射表予以修复；六是口语化的降压表述（如"降到"）未被识别为 DC-DC 类别触发词，通过扩充关键词覆盖列表解决。上述缺陷在修复后全部通过回归测试验证，且此后的迭代未产生新的回归问题。

\section{eZ-PLM 实时 API 集成测试}

在配置有效 API 密钥的环境下，以"12\,V 转 5\,V，3\,A，工业级"为输入执行端到端查询。系统共遍历了 9 个制造商型号前缀（TPS54、TPS62、LM2596 等），发起 9 次 API 请求，成功返回 31 条 TI 在产器件——型号涵盖 TPS54020RUWR、TPS54220DDAR、LM2596S 系列等，封装覆盖了 WSON、HTSSOP 和 D/N 等多种形态。五维过滤后全部有效器件的输入电压范围均满足 4.5$\sim$36\,V 的要求，输出电流均不低于 3\,A，温度范围均覆盖 $-40\sim85\;$°C，验证了 HMAC-SHA256 签名鉴权、多前缀分组查询和 MPN 电压推断链路的端到端正确性。

\section{评测局限性}

当前评测仍存在若干已知局限。在评分维度方面，LLM 增强评分所依赖的参考设计案例数据因 eZ-PLM 平台覆盖范围的限制，部分器件未能获取到关联的参考设计，此时系统自动回退为纯规则双维评分，LLM 软评分维度（应用适配度与设计成熟度）未能对全部器件生效。在报告质量方面，三类工程文档的质量评估目前采用人工定性检查，尚未建立自动化评测指标（如规范覆盖率、公式计算准确率）。在供应链数据方面，API 实时测试中部分器件的库存字段为空值（API 未返回 stock 数据），导致这些器件的供应链分被设为缺省中性值 50，未能完全反映真实的供应状态。
