\section{研究背景与问题分析}

电子元器件选型是电子产品研发流程中的关键环节。以DC-DC电源设计为例，工程师需要从数十家制造商的海量型号中，综合考虑输入/输出电压范围、输出电流、开关频率、封装尺寸、温度等级、车规认证、库存水平、价格、交期等十余个维度的参数约束，同时还需查阅数据手册验证设计公式、评估供应链风险。传统人工选型流程通常需要2小时以上，且存在以下瓶颈：

\begin{enumerate}[label=(\arabic*)]
  \item \textbf{知识检索碎片化}：设计公式、应用笔记、认证标准分散在数百份PDF文档和网页中，检索效率低；
  \item \textbf{供应链评估缺失}：选型时往往只关注电气参数匹配，忽略库存、生命周期、供应商集中度等供应链风险维度；
  \item \textbf{输出格式不标准}：选型结果以非结构化笔记形式记录，无法直接导入PLM系统形成可追溯的设计决策链；
  \item \textbf{国产替代困难}：在中美贸易摩擦背景下，工程师需要快速评估进口器件的国产替代可行性，但缺乏系统化的交叉对比工具。
\end{enumerate}

\section{技术难点}

本作品需要解决以下核心技术难点：

\subsection{自然语言需求的精确解析}

电子工程师的选型需求以自然语言形式输入，句式多变。例如“12V转5V 3A 车规”与“需要宽输入4.5-36V的降压方案，输出5V/3A，耐温-40到85度”表达的是相似的需求，但在电压范围表述、温度格式、关键词语序上存在显著差异。如何设计鲁棒的解析策略，既利用LLM的语义理解能力，又通过规则兜底保证关键参数的提取准确性，是本作品的第一个难点。

\subsection{多维度知识融合与推理}

选型决策需要融合三类异构知识：（1）器件规格参数的结构化数据（来自eZ-PLM API）；（2）设计公式和工程规范的文本知识（来自数据手册和标准文档）；（3）参考设计案例的半结构化数据。如何将这三种知识有机融入同一个推理链条，在评分、风险评估、拓扑分析等环节中产生协同增益，是第二个难点。

\subsection{Agent工具编排与对话管理}

系统不仅需要完成单次选型分析，还需支持多轮追问场景。例如工程师先搜索降压方案，再追问“第一款推荐有国产替代吗”。Agent需要维护对话上下文，在合适的时机自主选择调用搜索工具、知识检索工具或替代查找工具。工具调用过多会导致延迟增加和Token消耗，过少则无法获取足够信息。如何在准确性与效率之间取得平衡，是第三个难点。

\subsection{标准化输出与平台对接}

eZ-PLM平台要求输出结构化的PartIR、RiskIR和TopologyIR数据。需要设计合理的Pydantic数据模型，既完整描述电路拓扑的节点关系、外围元件参数、热性能估算，又保持模型的可扩展性以适配不同拓扑类型（Buck/Boost/LDO）。

\section{主要创新点}

\begin{enumerate}[label=(\arabic*)]
  \item \textbf{LLM优先+规则兜底的双重解析机制}：设计四级解析策略——LLM语义解析→规则化Category/Topology覆盖→电压比较兜底→温度范围模式匹配，确保关键参数100\%准确提取。支持mA单位转换、否定句式（如“非车规”）识别、无°C后缀的温度范围匹配（如“-40到85°C”）；
  \item \textbf{RAG知识增强的工程推理}：构建基于ChromaDB+sentence-transformers的工程知识向量库，将电感选型公式、环路补偿方法、AEC-Q100认证要求、PCB Layout规范等12类专业知识以语义检索方式注入选型决策链。在BOM清单和拓扑分析报告中显式标注知识来源与相关度评分；
  \item \textbf{五维加权评分模型}：突破传统纯参数匹配的局限，引入供应链稳定性、应用场景适配度（LLM评估）、设计成熟度和证据置信度四个额外维度。支持纯规则（80\%参数+20\%供应）与LLM增强（40\%参数+10\%供应+25\%应用+25\%设计）双模式自适应切换；
  \item \textbf{Pipeline+Agent双模架构}：提供线性Pipeline（适用于API集成/批量处理）和ReAct Agent（适用于交互式多轮对话）两种运行模式。Agent模式基于LangChain 1.3框架，配置4个专用工具，支持会话级状态管理；
  \item \textbf{三层IR标准化输出}：定义PartIR（器件信息）、RiskIR（风险条目+FMEA量化）、TopologyIR（拓扑节点图+外围元件BOM+热估算+Layout规则）三套Pydantic数据模型，输出可直接序列化为JSON对接eZ-PLM平台。
\end{enumerate}
