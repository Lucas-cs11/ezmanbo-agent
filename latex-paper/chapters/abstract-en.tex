% English Abstract

Electronic component selection remains a labor-intensive and risk-concentrated bottleneck in hardware development lifecycles, where engineers must verify multi-dimensional constraints across thousands of part numbers from dozens of manufacturers, typically consuming over two hours per design with no systematic supply chain risk identification or standardized engineering documentation output.

This paper presents an intelligent component selection and risk assessment Agent system for the eZ-PLM platform, built upon a five-stage linear pipeline---natural language requirement parsing, real-time API component retrieval, hybrid intelligent scoring, evidence chain construction, and standardized report generation---with an integrated LangChain ReAct Agent mode supporting multi-turn dialogue.

The primary technical contributions are fivefold. First, a manufacturer-prefix grouped query strategy is proposed to overcome the eZ-PLM API's limitation of supporting only MPN-prefix keyword search, achieving recall of over 200 active components per query across four major manufacturers (TI, ADI including LTC, Microchip, and ST). Second, a four-level fault-tolerant requirement parsing chain is designed, cascading LLM semantic parsing, rule-based category/topology override, voltage-comparison topology fallback, and multi-format temperature regex matching, which systematically resolves six categories of engineering parsing defects including milliampere unit mismatches, negation pattern failures, and LLM topology hallucinations. Third, an LLM hybrid scoring mechanism grounded by eZ-PLM reference design API data is introduced, yielding a four-dimensional weighted score: parameter matching (40\%), supply chain stability (10\%), application fitness (25\%), and design maturity (25\%), with automatic degradation to rule-only two-dimensional scoring when no reference design data is available. Fourth, a dual-layer risk assessment architecture separating rule-engine fact verification from LLM engineering narration is implemented, covering 13 deterministic risk detection categories with explicit data provenance labeling. Fifth, standardized intermediate representations (PartIR, ScoreBreakdown, TopologyIR, RiskIR) are defined, producing three engineering output documents---BOM selection list, supply chain risk assessment report, and circuit topology analysis report---in formats directly compatible with engineering review and eZ-PLM platform integration.

Experimental validation on 28 test cases (14 DC-DC Buck converters, 4 DC-DC Boost converters, and 10 LDO regulators) demonstrates a 100\% pass rate, with 100\% accuracy on all core constraint fields. The eZ-PLM real-time API integration test successfully returned 31 active TI components. A RAG engineering knowledge base (29 entries across 11 categories) and a ReAct Agent interaction mode further enhance the system's domain knowledge depth and interactive flexibility.
