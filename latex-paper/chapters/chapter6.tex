\section{工作总结}

本文面向硬禾科技旗下的 eZ-PLM 电子元器件全生命周期管理平台，设计并实现了一套将大语言模型（LLM）与工程规则引擎协同推理的智能元器件选型与风险评估 Agent 系统，完成了从自然语言需求输入到 BOM 选型清单、供应链风险评估报告和电路拓扑分析报告三类标准化工程文档输出的端到端自动化流程，同时以 Pipeline 与 ReAct Agent 双模架构兼顾了批量处理效率与交互式多轮推理的灵活性。

在 eZ-PLM 实时 API 集成方面，系统通过制造商型号前缀分组查询策略成功接入了 TI、ADI（含 LTC）、Microchip 和 ST 四大制造商的开放在产物料数据，以 HMAC-SHA256 签名认证机制保障了 API 调用的安全性，单次选型查询可召回超过 200 条真实在产器件，并通过五维过滤函数和基于 MPN 字符串的固定输出电压推断机制保障了检索结果的准确性。

在自然语言需求解析方面，系统设计并实现了四级容错需求解析链——LLM 语义解析、规则化类别与拓扑覆盖、电压比较方向兜底以及多格式温度正则匹配——系统性地修复了 mA 单位量级误差、否定句式漏判（"非车规"被正确映射为工业级）、LLM 拓扑幻觉（"5 V 转 3.3 V"被 LLM 误判为 LDO 的问题被电压比较兜底规则自动修正）等六类工程解析缺陷，在 28 条测试用例（14 条 DC-DC Buck 降压、4 条 DC-DC Boost 升压和 10 条 LDO）上取得了 100\% 的通过率，核心参数字段解析准确率达 100\%。

在混合智能评分方面，系统构建了以 eZ-PLM 参考设计接口数据为上下文约束的 LLM 软评分机制，实现了纯规则双维评分（参数匹配 80\% + 供应链稳定性 20\%）与 LLM 增强四维评分（参数匹配 40\% + 供应链稳定性 10\% + 应用适配度 25\% + 设计成熟度 25\%）两种模式的自适应切换，评分引擎集成了核心型号去重合并、动态约束归一化与 Top-N 推荐分级等工程优化，其中各维度的权重取值均依据了元器件选型与供应链风险评估领域的文献依据，去重合并策略以 LM2596 系列多封装变体为例验证了将同一型号族的展示冗余从"多条目罗列"压缩为"最优变体代表"的有效性。

在风险评估方面，系统实现了规则引擎事实验证与 LLM 工程叙述相分离的双层风险评估架构——规则层覆盖了 13 类确定性风险检测场景（包括无候选器件、库存低于阈值、停产生命周期、供应商集中度过高、关键数据字段缺失等），每条检测结果以结构化的 RiskItem 形式记录风险类型、严重程度、描述、相关器件型号和缓解建议，并与工业界失效模式与影响分析（FMEA）中"失效模式—失效影响—现行控制措施—建议措施"的记录范式保持一致；LLM 层仅被限定为对已验证的风险条目进行工程背景阐述，不拥有独立生成新风险结论的权限，报告中由 LLM 生成的叙述段落均被显式标注数据来源声明。

在标准化中间表示（IR）方面，系统以 Pydantic v2 定义了一套完整的 IR 数据模型体系，包括 PartIR（21 个字段）、ScoreBreakdown（含评分模式与 LLM 子分项）、TopologyIR（四子模型：拓扑功能节点图、外围元件参数表、热性能估算与 Mermaid 框图）和 RiskIR（含 FMEA 量化字段），三类 Markdown 工程文档与结构化 TopologyIR JSON 文件被同步生成，输出格式可直接服务于工程评审及 eZ-PLM 平台的数据互通。

在知识增强与智能交互方面，系统基于 ChromaDB 向量存储与 sentence-transformers 嵌入模型构建了包含 29 条工程知识条目、覆盖 11 个专业类别的 RAG 工程知识库，知识检索结果在每次 Pipeline 分析中被自动注入三份输出报告；同时基于 LangChain 框架的 ReAct 范式实现了包含四个专用工具的 Agent 交互模式，支持通过 \texttt{session\_id} 机制进行多轮对话，单轮选型场景下调用 2 次工具、总耗时约 8 秒，多轮追问场景下能够正确维护对话上下文中的推荐器件型号。

\section{不足与展望}

当前系统在以下方面仍存在可改进的空间。

在知识库规模方面，29 条工程知识条目仅覆盖了 Buck/Boost/LDO 的基础设计场景和常用工业标准，后续可通过自动文档分块与向量化摄入脚本批量导入 TI/ADI 应用笔记\cite{slva057,slva477b,snva538}、IPC 标准\cite{ipc2221,ipc2152}及 IEC 可靠性规范等专业文档，将知识条目规模扩展到 100 条以上，从而进一步丰富拓扑分析报告中设计公式的引用来源和风险评估报告中的标准依据。

在评测覆盖度方面，当前 28 条测试用例以 Buck 拓扑为主（14 条），Boost 和 LDO 的用例数量相对偏少，且尚未建立 MCU、传感器、连接器等更多元器件品类的测试用例集。此外，LLM 混合评分模式的定量对比基准——例如混合评分与纯规则评分的相关性与一致性分析——也有待建立，以更系统化地评估 LLM 软评分维度的实际增益。

在偏好解析方面，用户可能在自然语言中使用"库存充足""希望供货稳定""价格不要太贵"等隐式表达来描述选型偏好，这些表达同当前预定义偏好标签（domestic\_alternative、low\_supply\_risk 等）之间的映射关系还不够完善。后续可通过扩充同义词规则库或引入 LLM 辅助的偏好语义提取来提升偏好解析的覆盖面与准确率。

在前端展示方面，当前 Streamlit 界面已具备评分仪表盘、证据链面板和参数对比表格等核心功能，但尚未集成 Agent 推理步骤的实时可视化——即工具调用序列的流水线式展示、每个工具的输入参数与返回结果的折叠面板，以及多轮对话历史的时间轴视图。这一功能的实现将有助于竞赛评审专家更直观地理解系统的内部推理过程。

\section{应用价值}

本系统的工程实用价值主要体现在效率提升、决策质量改善和平台生态构建三个层面。

在效率层面，传统耗时两小时以上的人工选型流程被压缩至分钟级——API 调用加本地 Pipeline 处理的合计耗时约 1 分钟，其中本地处理环节的中位延迟仅为 20 毫秒——显著降低了工程师在参数核查、数据手册查阅和文档整理上的重复劳动，使得工程师可以将更多精力投入到方案对比、拓扑优化和测试验证等需要深度专业知识的高价值决策环节。

在决策质量层面，混合评分体系将 eZ-PLM API 的实时物料数据、参考设计库的结构化案例信息以及 LLM 的工程知识表达能力有机整合，生成的 BOM 选型清单和风险评估报告经工程师复核后可直接用于设计评审；RAG 知识增强使得拓扑分析报告中的电感与电容选型计算具备了可追溯到 TI 应用笔记或 IPC 标准的工程依据\cite{gao2023retrieval}；证据链与数据来源标注机制则保证了选型结论的可解释性和可验证性，满足了工程评审和质量管理流程对设计决策可追溯性的要求。

在平台生态层面，系统通过标准化的 IR 数据模型体系与 eZ-PLM 平台实现了深度集成——PartIR 可直接映射至平台的物料属性字段、RiskIR 为平台的风险管理模块提供了结构化的风险数据接口、TopologyIR JSON 可被平台的设计复用模块直接消费——为平台提供了一套可扩展的 AI 增强选型能力框架。Pipeline 与 Agent 双模架构兼顾了企业级批量选型任务的处理效率和面向个体工程师的交互式辅助决策的灵活性，在国产替代\cite{wang2022selection}和供应链多元化布局的战略背景下，为电子工程师群体提供了一种有据可查、可追溯、可验证的 AI 辅助选型工具。
